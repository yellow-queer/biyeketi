# 📚 RAG 知识库使用指南

## 🎯 功能概述

RAG（Retrieval-Augmented Generation）知识库系统支持多种格式文档的自动加载、向量化和智能检索。

## 📁 支持的文件格式

### ✅ 已支持格式

| 格式 | 扩展名 | 说明 | 依赖库 |
|------|--------|------|--------|
| **文本文件** | `.txt` | 纯文本文件 | 无需额外依赖 |
| **Markdown** | `.md` | Markdown 文档 | 无需额外依赖 |
| **PDF 文档** | `.pdf` | PDF 文件 | `PyPDF2` |
| **Word 文档** | `.docx` | Word 2007+ | `python-docx` |
| **图片文件** | `.jpg`, `.jpeg`, `.png` | 图片 OCR 识别 | `PaddleOCR` |

### ⚠️ 注意事项

- `.doc`（旧版 Word）需要额外配置
- 图片 OCR 首次使用会下载 OCR 模型（约 200MB）
- PDF 文件应包含可提取的文字（扫描件需要 OCR）

## 🔍 使用知识库检索

### 方法一：前端界面

1. 打开网站：http://127.0.0.1:5000
2. 在对话窗口点击 📚 按钮
3. 输入查询关键词
4. 查看检索结果

### 方法二：API 调用

```python
import requests

response = requests.post(
    'http://127.0.0.1:5000/api/rag/search',
    json={
        'query': '柑橘实蝇检测方法',
        'top_k': 3
    }
)

results = response.json()['results']
for result in results:
    print(f"文档：{result['name']}")
    print(f"相似度：{result['score']:.3f}")
    print(f"内容：{result['content'][:100]}...")
```

### 方法三：Python 代码

```python
from RAG.knowledge_base import get_knowledge_base

kb = get_knowledge_base()
results = kb.search('柑橘实蝇检测方法', top_k=3)

for result in results:
    print(f"{result['name']}: {result['content']}")
```

## 💡 最佳实践

### 文档准备

1. **文本质量**
   - 确保文字清晰可读
   - PDF 应为文字版（非扫描件）
   - 图片应包含足够文字信息

2. **文档组织**
   - 按主题分类命名
   - 避免过大文件（建议 < 10MB）
   - 定期清理过时文档

3. **内容优化**
   - 使用结构化格式（标题、段落）
   - 包含关键词和术语
   - 避免过多格式标记

### 检索优化

1. **查询技巧**
   - 使用具体关键词
   - 避免过长句子
   - 尝试同义词

2. **参数调整**
   - `top_k`: 返回结果数量（默认 3）
   - 增加数量可提高召回率

3. **索引管理**
   - 定期重建索引
   - 监控索引大小
   - 清理无用文档

## 🐛 常见问题

### Q: 文档未被加载？

**A:** 检查以下几点：
1. 文件扩展名是否在支持列表中
2. 文件是否为空
3. 查看控制台错误信息
4. 确认编码为 UTF-8

### Q: PDF 读取失败？

**A:** 
```bash
# 安装 PyPDF2
pip install PyPDF2

# 如果是扫描件，需要 OCR
pip install paddleocr
```

### Q: Word 文档无法读取？

**A:** 
```bash
# 安装 python-docx
pip install python-docx

# 注意：仅支持 .docx 格式
# .doc 格式需要转换
```

### Q: 图片 OCR 很慢？

**A:** 
- 首次使用会下载模型（耐心等待）
- 建议使用 GPU 版本
- 减少图片数量或分辨率

### Q: 检索结果为空？

**A:** 
1. 确认文档已成功加载
2. 检查查询关键词是否匹配
3. 尝试增加 `top_k` 值
4. 重建索引

### Q: 内存不足？

**A:** 
1. 减少文档数量
2. 使用 CPU 模式
3. 分批处理文档
4. 增加系统内存

## 🎓 技术原理

### 文档向量化

1. **文本提取**：从各种格式文件中提取文字
2. **分词处理**：使用 Sentence Transformers 分词
3. **向量编码**：将文本转换为 384 维向量
4. **索引构建**：使用 FAISS 构建向量索引

### 相似度检索

1. **查询向量化**：将用户查询转换为向量
2. **相似度计算**：计算查询向量与文档向量的距离
3. **排序返回**：按相似度排序返回 top_k 结果

### 更新检测

1. **内容哈希**：为每个文档计算 MD5 哈希
2. **定期比对**：检查文档是否变化
3. **增量更新**：只更新变化的文档
